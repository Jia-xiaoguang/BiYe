# 无人机围栏异常巡检系统总体架构建议

## 1. 感知采集层

### 主要组成
- 无人机机载相机
- 视频流接入模块
- 本地视频导入模块
- 巡检任务元数据采集模块

### 主要功能
- 获取无人机实时巡检视频流
- 支持离线视频文件导入
- 记录巡检任务编号、时间、区域、设备编号等基础信息

> 该层解决“数据从哪里来”的问题。

## 2. 边缘智能分析层

该层是系统核心，建议包含以下三个模块。

### 2.1 视频解析与调度模块

#### 功能
- 视频解码
- 按帧读取与缓存管理
- 实时流与离线流统一调度
- 帧率控制与推理任务分发

#### 作用
- 为 LSENet 图像增强和异常检测模块提供标准化输入。

### 2.2 图像增强模块（LSENet）

#### 功能
- 对巡检视频中的运动模糊帧进行去模糊增强
- 输出清晰图像供后续检测使用
- 提升围栏边缘、纹理与异常局部细节可辨识度

#### 系统定位
该模块不是孤立的图像增强单元，而是后续异常检测的前置支撑模块。

#### 建议论文描述
为了兼顾检测精度与边缘计算效率，系统在智能分析层中集成 LSENet 图像增强模块。该模块针对无人机巡检过程中由于飞行抖动、平台运动及视角变化导致的图像运动模糊问题，对输入视频帧进行轻量化去模糊恢复，从而提高围栏边缘结构与异常局部细节信息的清晰度，为后续异常检测提供高质量视觉输入。

### 2.3 异常检测与轨迹一致性验证模块

该模块由以下两个子模块组成：
- FAD-YOLOv11 异常检测子模块
- DeepSORT 轨迹一致性验证子模块

#### 功能
- 检测围栏破损、倾倒、缺失、人员翻越等异常
- 输出异常类别、边界框与置信度
- 对连续帧目标进行关联跟踪并分配统一轨迹 ID
- 利用轨迹持续性与一致性抑制误检
- 输出更稳定的视频级告警结果

#### 系统价值
系统不仅做“单帧检测”，还通过跨帧一致性判定异常是否持续存在，从而减少瞬时误检与检测抖动。

#### 建议定义
该模块是系统完成围栏异常智能识别的核心处理模块，负责实现异常目标的帧级检测、跨帧关联以及轨迹级一致性验证。通过将 FAD-YOLOv11 的高效检测能力与 DeepSORT 的时序关联能力相结合，系统能够实现从单帧异常发现到视频级稳定识别的统一建模。

## 3. 业务应用层

### 3.1 实时巡检监控模块
- 显示无人机实时视频画面
- 叠加异常检测框、类别标签、置信度、轨迹 ID
- 展示系统运行状态与巡检任务信息

### 3.2 异常告警模块
- 在轨迹一致性判定满足条件后触发告警
- 显示异常类型、时间、位置、持续帧数等信息
- 支持异常截图保存与事件留档

> 建议突出“告警由轨迹一致性验证触发，而非单帧检测直接触发”。

### 3.3 历史记录与结果回放模块
- 查询巡检历史任务
- 查看异常事件记录
- 回放对应异常视频片段
- 按异常类型、时间、区域检索结果

### 3.4 模型配置与运行控制模块
- 配置是否开启 LSENet 去模糊
- 配置检测置信度阈值
- 配置轨迹一致性阈值
- 配置告警触发规则
- 启动/停止实时巡检任务

> 可用于展示开启/关闭去模糊、轨迹一致性验证前后的识别稳定性差异。

## 4. 数据管理层

### 4.1 视频与图像数据
- 原始巡检视频
- 去模糊增强帧
- 异常截图
- 回放索引

### 4.2 检测结果数据
- 帧号
- 异常类别
- 边界框坐标
- 置信度
- 轨迹 ID
- 最终告警标记

### 4.3 任务与设备数据
- 巡检任务编号
- 无人机编号
- 巡检区域
- 巡检时间
- 操作员信息
- 系统运行日志
